# OpenClaw + GLM-5 200K 上下文配置优化指南

> 来源：飞书文档
> 日期：2026-03-07

---

## 📋 文档信息

- 适用场景：GLM-5 200K 上下文 + 飞书通道 + Memory Pro 插件
- 问题类型：配置错误、性能优化、上下文管理

---

## 🔴 问题诊断

### 问题 1: 配置文件语法错误导致重启失败

**现象：**
```
Config invalid: JSON5 parse failed: SyntaxError: JSON5: invalid character at 139:1 Invalid input (allowed: "default", "safeguard")
```

**根本原因：** agents.defaults.compaction.mode 被设置为 "auto"，OpenClaw 只支持 "default" 和 "safeguard" 两种模式

---

### 问题 2: Memory Pro 插件配置不规范

**现象：**
```
plugins.entries.memory-lancedb-pro.config: invalid config: must NOT have additional properties
```

**错误参数：**
- maxRecallItems ❌
- recallThreshold ❌
- sessionStrategy ❌
- autoCompaction ❌
- compactionThreshold ❌

**正确参数：**
- autoRecallMinLength ✅
- retrieval.hardMinScore ✅
- sessionMemory.enabled ✅

---

### 问题 3: GLM-5 200K 上下文未充分利用

**现象：** 默认配置未针对 GLM-5 优化，上下文边界设置过高或过低

---

## 💭 思考分析

### 1. Compaction 模式选择

| 特性 | default | safeguard |
|------|---------|----------|
| 压缩策略 | 增量式压缩 | 分块摘要压缩 |
| 触发时机 | 正常节奏 | 更激进 |
| 适用场景 | 短对话 | 长会话、复杂任务 |
| 大上下文处理 | 可能失败 | 优化处理 |

**决策：** 选择 safeguard 模式

---

### 2. 上下文边界计算

**GLM-5 理论窗口：** 204,800 tokens

**安全边界考量：**
- 系统提示词: ~2,000 tokens
- 工具定义: ~5,000 tokens
- 工具调用 + 响应: ~10,000 tokens
- 临时占用: ~5,000 tokens

**计算：**
```
204,800 - 2,000 - 5,000 - 10,000 - 5,000 = 182,800 tokens
```

**安全系数 0.85 → 155,380 tokens**

**最终选择：** 156,000 tokens (156K)

---

### 3. 飞书历史消息平衡

| 类型 | 历史消息数 |
|------|----------|
| 群聊 | 50 条 |
| 私聊 | 30 条 |

---

### 4. Memory Pro 插件优化

| 参数 | 值 | 作用 |
|------|-----|------|
| autoCapture | true | 自动捕获重要信息 |
| autoRecall | true | 自动召回相关记忆 |
| autoRecallMinLength | 6 CJK | 最小触发长度 |
| retrieval.mode | hybrid | 混合检索（向量+关键词）|
| retrieval.hardMinScore | 0.72 | 相似度阈值 72% |
| sessionMemory.enabled | true | 会话总结存储 |

## 💡 经验总结

### 1. 配置验证经验
- ❌ 直接修改配置不验证 → 启动失败
- ✅ 修改后立即 `openclaw config validate` → 快速发现问题

### 2. 插件配置经验
- ❌ 随意添加参数 → "must NOT have additional properties"
- ✅ 按 schema 配置 → 配置有效

### 3. 上下文边界经验
- 理论窗口: 204,800 tokens
- 安全边界: ~48,800 tokens (24%)
- 有效上下文: 156,000 tokens (76%)

### 4. Compaction 模式经验
- 短对话，快速响应 → default
- 长会话，复杂任务 → safeguard
- ✅ GLM-5 200K → safeguard

### 5. 历史消息经验
- 低密度 (群聊、闲聊): 50-100 条
- 中密度 (私聊，工作): 30-50 条
- 高密度 (技术讨论): 20-30 条

### 6. 飞书插件消息处理经验
- Bot mention 自动清除 ✅
- 用户 mention 保留为 @用户名 ✅
- 富文本转 Markdown ✅
- 附件转资源引用 ✅

### 7. 重启 Gateway 经验
- ❌ `taskkill /F /IM node.exe` 会杀掉当前会话
- ✅ 在新窗口启动: `start "OpenClaw Gateway" cmd /c "openclaw gateway"`

### 8. 配置文件保护经验
- 禁用 /config 命令: `"commands": { "config": false }`

## 📄 核心配置 JSON

```json
{
  "agents": {
    "defaults": {
      "contextTokens": 156000,
      "compaction": {
        "mode": "safeguard",
        "reserveTokensFloor": 40000
      }
    }
  },
  "channels": {
    "feishu": {
      "historyLimit": 50,
      "dmHistoryLimit": 30
    }
  }
}
```
